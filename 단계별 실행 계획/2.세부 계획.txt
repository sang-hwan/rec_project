### 📌 **2단계 세부 실행 계획**

**1. 매일 자동 데이터 수집 스케줄 설정**

* 데이터 수집을 위한 최적의 시간대를 한국 기준 오후 5시에서 7시 사이로 지정합니다.
* 서버 또는 개인 컴퓨터에 설정한 자동화 프로그램을 크론(cron) 또는 작업 스케줄러로 등록하여, 매일 지정된 시간에 정확히 실행되도록 합니다.
* 자동 실행이 정상 작동하는지 하루 동안의 테스트 실행을 통해 확인합니다.

**2. 데이터 수집 범위 설정**

* 매일 수집할 데이터는 팔로우 중인 주요 계정의 최근 24시간 동안 게시된 모든 트윗으로 설정합니다.
* API 호출의 제한 조건을 고려하여, 각 계정당 하루의 트윗을 빠짐없이 가져올 수 있도록 호출 방식을 최적화합니다.
* 특히 중요한 계정이나 인플루언서의 트윗은 별도의 태그 또는 라벨로 구분하여 중요도를 강조합니다.

**3. 수집된 데이터의 저장 및 관리**

* 수집된 데이터를 날짜별로 구분하여 데이터베이스, Notion 또는 구글 시트 등의 문서 도구에 자동으로 정리되도록 합니다.
* 날짜, 계정명, 계정의 카테고리(예: 주식 전문가, 기관 투자자 등), 트윗 본문, 게시 시각, 좋아요 수, 리트윗 수 등의 핵심 정보를 깔끔하게 정리합니다.
* 저장 구조를 일관되게 유지하여 분석 및 조회가 편리하게 구성합니다.

**4. 데이터 수집 모니터링 및 예외 처리**

* 데이터 수집이 매일 원활하게 진행되는지 주기적으로 확인하는 모니터링 루틴을 설정합니다.
* API 호출 제한이나 네트워크 오류 등으로 데이터 수집이 실패했을 때 자동으로 알림을 받을 수 있도록 이메일이나 메시지 알림 기능을 설정합니다.
* 데이터 누락이나 중복이 발생하지 않도록 수집된 데이터를 주기적으로 검수합니다.

**5. 자동 요약 시스템 구축 준비**

* 수집된 원본 트윗 데이터에서 요약할 주요 정보를 정의합니다. (예: 핵심 키워드, 자주 언급된 주식 종목, 경제 전망 등)
* GPT API 또는 다른 요약 AI 모델을 선정하고, 사용 환경을 구성합니다.
* 자동 요약을 위해 AI 모델의 입력 형식과 출력 형식을 명확하게 설정하여 자동화 프로그램과 연결이 쉽게 합니다.

**6. 요약 시스템 초기 테스트 및 보정**

* 수집된 트윗 데이터를 바탕으로 자동 요약 시스템의 성능을 테스트합니다.
* 요약 결과의 품질을 평가하고, 요약 결과가 정확하지 않거나 부족한 경우 입력 데이터를 조정하거나 요약 모델의 설정을 세부적으로 수정합니다.
* 테스트 기간(2\~3일)을 두어 품질을 충분히 검증하고, 요약 시스템의 신뢰성을 확보합니다.

**7. 데이터 정리 및 저장 형태 최종화**

* 자동 요약된 데이터와 원본 데이터를 구분하여 별도의 정리 공간에 함께 저장하여 관리합니다.
* 정리된 데이터를 쉽게 접근하고 조회할 수 있도록 구조화하고, 사용자가 원하는 날짜와 계정별로 검색 가능하도록 합니다.
* 최종적으로 자동화된 데이터 수집-요약-저장 시스템이 문제없이 연속으로 작동하도록 유지보수 체계를 구축하여 안정화합니다.
